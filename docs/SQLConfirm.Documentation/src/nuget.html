SQLConfirm - Documentation

NuGet Packages

There are currently 6 NuGet packages, and 1 NuGet tool.

Most developers will only need to install the SQLConfirm.Packages.* NuGet packages, since these contain everything you need to start testing.

SQLConfirm.Packages.*

SQLConfirm currently integrates with MSTest and NUnit as testing frameworks, and SQL Server as database engine.

Therefore, there are 2 packages to choose from, one for MSTest and NUnit. Install the one that you want, and you'll have everything you need.

  • For integrating with MSTest and SQL Server, use SQLConfirm.Packages.SQLServer.MSTest
  • For integrating with NUnit and SQL Server, use SQLConfirm.Packages.SQLServer.NUnit

SQLConfirm.Core

SQLConfirm.Core is the core library that handles all the SQLConfirm logic. This is common logic across all test frameworks and database engines.

  • SQLConfirm.Core

SQLConfirm.Frameworks.*

SQLConfirm must run with a testing framework, currently either MSTest or NUnit. These SQLConfirm.Frameworks.* packages contain the integration logic.

  • For integrating with MSTest, use SQLConfirm.Frameworks.MSTest
  • For integrating with NUnit, use SQLConfirm.Frameworks.NUnit

SQLConfirm.Databases.*

SQLConfirm currently can only use SQL Server as a database engine. This package handles the logic to insert and retrieve data from the database.

  • For integrating with SQL Server, use SQLConfirm.Databases.SQLServer

SQLConfirm.TemplateGeneration

There is a TemplateGeneration tool that can be used to automatically generate template files based on the schema of your database tables.

This tool outputs ready-to-go classes directly into your test project, referencing all columns that exist in the tables.

  • To run the tool, use SQLConfirm.TemplateGeneration
